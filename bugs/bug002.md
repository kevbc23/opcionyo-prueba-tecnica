# Bug 2 — Usuario puede acceder a información de una cita perteneciente a otro paciente

## Qué pasa

El sistema no valida correctamente los permisos del usuario autenticado sobre el recurso solicitado, permitiendo potencialmente consultar información de una cita que pertenece a otro paciente.

## Cómo reproducirlo

### Precondiciones

- Existen al menos dos pacientes registrados en el sistema.
- El paciente `PATIENT-001` cuenta con una cita creada.
- El paciente `PATIENT-002` tiene una cuenta activa y puede autenticarse.

Ejemplo:

```text
Paciente propietario de la cita:
PATIENT-001

Cita registrada:
AppointmentId: APT-1001
```
### Pasos
1. Iniciar sesión con el usuario PATIENT-002.
2. Obtener o conocer el identificador de una cita perteneciente a otro paciente.
3. Consultar la información de la cita utilizando directamente su identificador.

Ejemplo:

```text
GET /appointments/APT-1001
```
4. Revisar la respuesta obtenida.

### Resultado actual
El sistema retorna información de la cita solicitada:
```http
HTTP 200 OK
```
La respuesta contiene información asociada a un paciente diferente al usuario autenticado.

### Resultado esperado
El sistema debería validar que el usuario autenticado tenga autorización para consultar la cita solicitada.

Si la cita pertenece a otro paciente, la solicitud debería ser rechazada:
```http
HTTP 403 Forbidden
```
Ejemplo:
```json
{
  "error": "Acceso denegado. No tiene permisos para consultar esta cita."
}
```

### Severidad

Crítica

### Impacto

Puede ocasionar:

Puede ocasionar exposición de información sensible, como:

- Datos personales de pacientes.
- Información relacionada con citas médicas.
- Datos privados asociados a la atención del usuario.
