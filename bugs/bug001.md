# Bug 1 — Permite crear una cita con fecha pasada

## Qué pasa

El sistema permite registrar una cita utilizando una fecha anterior al día actual, generando información inconsistente dentro del calendario de atención.

## Cómo reproducirlo

### Precondiciones

- Usuario autenticado correctamente como paciente.
- Especialista disponible para agendamiento.

### Pasos

1. Ingresar al flujo de creación de una cita.
2. Enviar una solicitud utilizando una fecha anterior a la fecha actual.

Ejemplo:

```json
{
  "patientId": "PATIENT-001",
  "specialistId": "DOC-001",
  "appointmentDate": "2025-01-10",
  "time": "10:00"
}
```
3. Procesar la solicitud.

### Resultado actual
El sistema acepta la solicitud y crea la cita correctamente:
HTTP 201 Created
La información queda registrada con una fecha que ya pasó.

### Resultado esperado
El sistema debería validar que la fecha de la cita sea igual o posterior al día actual.
La respuesta del servidor debe ser: HTTP 400 Bad Request

### Severidad

Media

### Impacto

Puede ocasionar:

- Registros inválidos dentro del historial de citas.
- Información incorrecta en reportes y métricas.
- Problemas en la planificación de disponibilidad del especialista.
- Confusión para usuarios que visualicen citas fuera de contexto temporal.
