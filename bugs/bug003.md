# Bug 3 — Permite cancelar una cita que ya fue cancelada

## Qué pasa

El sistema permite ejecutar nuevamente la operación de cancelación sobre una cita que ya se encuentra cancelada.

## Cómo reproducirlo

### Precondiciones

- Existe una cita creada con estado confirmado.

Ejemplo:

```json
{
  "appointmentId": "APT-1001",
  "status": "CONFIRMED"
}
```

### Pasos

1. Ejecutar la cancelación de la cita.

Ejemplo:

```http
DELETE /appointments/APT-1001
```

2. Verificar que la cita cambia al estado cancelado.

Respuesta:

```json
{
  "appointmentId": "APT-1001",
  "status": "CANCELLED"
}
```

3. Ejecutar nuevamente la misma solicitud de cancelación:

```http
DELETE /appointments/APT-1001
```

4. Revisar la respuesta obtenida.

### Resultado actual

El sistema permite cancelar nuevamente una cita que ya se encuentra en estado cancelado:

```http
HTTP 200 OK
```

La operación se procesa como si fuera una nueva cancelación.

### Resultado esperado

El sistema debería validar el estado actual de la cita antes de procesar la solicitud.

Al intentar cancelar una cita que ya fue cancelada, debería responder:

```http
HTTP 409 Conflict
```

Ejemplo:

```json
{
  "message": "La cita ya se encuentra cancelada"
}
```

### Severidad

**Media (Medium)**

### Impacto

Puede ocasionar:

- Ejecución innecesaria de procesos asociados a la cancelación.
- Envío duplicado de notificaciones al paciente o especialista.
- Problemas de trazabilidad en el historial de cambios de la cita.
